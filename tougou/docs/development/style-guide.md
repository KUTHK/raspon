# コーディング規約 (Assembly Style Guide)

Goal: 可読性と一貫性を確保し保守容易性を高める。

## 命名

- ラベル: snake_case or CamelCase (統一ルールを別途決定)
- 定数マクロ: UPPER_SNAKE

## コメント指針

- 目的 / 副作用 / レジスタ使用を前置ブロックで
- 難解命令行の右側に短い説明

## レジスタ使用 (仮)

- r0-r3: パラメータ & 戻り値
- r4-r11: callee 保存
- sp / lr / pc: ARM 標準規約

## ファイル構造

1. ヘッダコメント (概要/依存)
2. 定数 / マクロ
3. データ領域
4. コード
